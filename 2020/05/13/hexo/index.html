Hexo个人博客的搭建

离开校园，愈发地感到时光易逝，一天能做的真的很少，只有每天持之以恒，才能不负韶华，有所成长，所以才想以博客的方式记录自己一路的辛酸与收获。

个人博客的搭建并不是什么技术活，小白也能按照🐏哥codesheepB站教学视频一步步搭建起自己的博客。写这篇博客我只想简单记录一下自己搭建的过程以及踩过的坑，顺便作个简单的思考与总结。

1.准备工作

首先，下载以下工具：

  • Node.js   Node.js 是一个基于 Chrome V8 引擎的 JavaScript 运行时环境，类似于jre吧可能。下载地址
  • git   git是一个开源的强大的分布式版本控制系统，也就是帮你管代码的系统。下载地址

Node.js下载是包含npm包管理器的，通过npm你可以下载，上传js代码，这里主要通过npm下载js包。

2.开始搭建

Node.js和git下载成功后，打开git bash也就是git的命令行窗口，输入以下命令

node -v       //查看node.js安装版本
npm -v        //查看npm版本
npm install -g cnpm --registry=http://registry.npm.taobao.org    //由于npm安装插件是从国外服务器下载，下载速度很慢，要先下载淘宝镜像cnpm，从cnpm再下载插件
cnpm -v      //查看cnpm版本
cnpm install -g hexo-cli   //全局下载hexo框架
hexo -v

接着你可以在电脑磁盘里任意位置新建一个文件夹，名字随意，我的是blog文件夹，然后右键点击这个blog文件夹，点击Git Bash here,进入git的命令行窗口。或者你也可以接着上面的命令行窗口，cd到你要创建文件夹的目录，mkdir 文件夹名字。在命令行窗口输入以下指令。

hexo init       //初始化hexo，hexo在当前目录下自动生成一系列初始文件
hexo generate       //生成静态文件，也可以简化为hexo g
hexo server       //启动本地博客服务，默认为4000端口，也可简化为hexo s

开启服务后，打开浏览器访问localhost:4000，就可以显示hexo的默认页面。到这第一步基本算完成了。

3.在gitPages上部署网站

刚才网页的静态资源都是存放在自己本地的，要想让别人也能访问你的个人网站，必须将网站部署到服务器上。现在有很多云服务器，阿里云，腾讯云等等，不过这些都要收取一定的费用，虽然也不贵，10rmb每月，但是反正又没人看自己的博客，都是写给自己看的，花那个钱干嘛，别问，问就是穷！而在github上部署是免费的，但是访问速度很慢，而且域名死板，买服务器可以自己建域名。

  • 创建github仓库 ：注册一个github账号，然后新建一个仓库，名字必须为你注册的用户名.github.io
  • 下载插件：git bash输入以下指令git部署插件
    cnpm install --save hexo-deployer-git    //--save就是将对git插件的依赖自动保存到hexo的配置文件package.json中
  • 写入配置 ：打开bolg文件夹下的_config.yml文件，找到deploy配置，添加以下信息,username为你自己的github用户名，冒号后面有一个空格
    type: git
repo: https://github.com/username/username.github.io.git    //仓库名
branch: master    //分支名
  • 部署网站 ：清除缓存，重新生成静态资源，将public文件夹下的所有内容全部同步到github下的master分支下
    hexo clean
hexo g
hexo deploy
    ok,到这步基本算部署完成，至于博客的美化，主题的修改，博文的发表，标签和分类的生成，且听下回分解。
