web.xml配置说明

参考博客

1.作用

web.xml时=是JavaWeb中一个重要的配置文件，可以通过web.xml来配置servlet，filter，首页等。但它不是Javaweb所必需的。

2.web.xml头部

<?xml version="1.0" encoding="UTF-8"?>
<web-app xmlns="http://xmlns.jcp.org/xml/ns/javaee"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://xmlns.jcp.org/xml/ns/javaee http://xmlns.jcp.org/xml/ns/javaee/web-app_4_0.xsd"
         version="4.0">
</web-app>

一个xml文档的结构或者说是模式（schema）是需要定义的，只有引入了xsi，才能知道定义了哪些标签以及标签的用法。根标签web-app中引入两个命名空间，一个为默认命名空间，一个为xsi，后面的值为一个url作为唯一标识，在schemaLocation中默认命名空间的schema地址。

3.加载过程

WEB容器的加载顺序是：ServletContext -> context-param -> listener -> filter -> servlet。并且这些元素可以配置在文件中的任意位置。
加载过程顺序如下：

  1. 启动一个WEB项目的时候，WEB容器会去读取它的配置文件web.xml，读取<listener>和<context-param>两个结点。
  2. 紧急着，容器创建一个ServletContext（servlet上下文），这个web项目的所有部分都将共享这个上下文。
  3. 容器将<context-param>转换为键值对，并交给servletContext。
  4. 容器创建<listener>中的类实例，创建监听器。
