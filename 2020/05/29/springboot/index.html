SpringBoot学习

SpringBoot使得程序员摆脱了烦人的xml配置，而仅仅专注于业务代码的编写，极大地提高了开发的效率，这也是如今SpringBoot这么流行的原因。SpringBoot内置了一个轻量级的Tomcat，不需要再配置Tomcat，只需要运行Application类就行。懒得再写一步步怎么做，没啥意思，就写一些自己学到的和不足的地方。

1.Spring bean创建

在Spring中，bean可以被定义为两种模式：prototype（多例）和singleton（单例），Spring bean 默认是单例模式。一个pojo要被bean创建，首先要被加入container中，要被注释为component。无论你需不需要创建对象，都会contex初始化时都会创建容器内的对象。

public static void main(String[] args) {
        ApplicationContext context=SpringApplication.run(Springbootdemo2Application.class, args);
        Person p=context.getBean(Person.class);
        p.setLastName("xie");
        Person p2=context.getBean(Person.class);
        System.out.println(p2.getLastName());//xie
        System.out.println(p==p2);//true
    }

要想改为多例模式，pojo上加上另一个注释@Scope(value =”prototype” )

@Component
@Scope(value ="prototype" )
public class Person {
    private String lastName;
    private Integer age;
    private boolean boss;
    private Map<String,Object> map;
    private List<Objects> lists;
    private Dog dog;

    public Person(){
        System.out.println("person created...");
    }

    public String getLastName() {
        return lastName;
    }

2.@AutoWired

一个类依赖于另一个类，你创建了这个类，如果没有自动关联到另一个类，调用相关方法便会空指针异常，因为另一个相关类并没有被创建

@Autowired
 private Dog dog;

autowired是根据类型从container中搜索，每个pojo类通过@component加入到container中都会有class对象；你也可以通过名字搜索，指定pojo类在容器中的名字，然后用@Qualifier去搜索

@Autowired
   @Qualifier("d")
   private Dog dog;
@Component("d")
public class Dog {
    private String name;
    private int age;
}

3. hello.jsp

要访问一个hello.jsp，要有一个url映射到一个controller中对应方法，用@RequestMapping，
注释了@ResponseBody的方法返回的都是作为response的数据，如果要返回jsp或者img或其他文件，就不能标注这个注释。springboot默认是不支持jsp的，你浏览器访问jsp不会跳转而是会下载jsp。所以要引入tomcat-jasper依赖，直接用maven进行install就行，这里百度了半天的答案，全是尼玛坑壁解答，骗人真的很好玩么。好多都说要导入tomcat-embed-jasper，还说要导入javax-servlet，搞了半天一个没用，心累。早知道就直接听那个油管主的了。maven下载好依赖库后似乎并不会直接导入进来，要重启一下才有，也就是你本地仓库里是有的，但是你工程依赖库里并找不到，我后面将json转化为xml引入Jackson-dataformatter-xml时遇到了这个情况。
